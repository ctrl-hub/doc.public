<Info>**Target Personas:** Contractor Administrator, Training Manager, Workforce Manager</Info>

## Step-by-Step Guide

### Assign Members of Your Workforce to a Job Role

1. In the Ctrl Hub Web Console, navigate to **Competency &gt; Job Roles**
2. Select a Job Role from the list
3. Select **Assign someone to this role**
4. Find one or more people to assign and select **Assign users**

<Info>The selected users are added to the **Job Role &gt; Assigned &gt; Table** list immediately.</Info>

---

### Remove a User from a Job Role

1. From the **Job Role &gt; Assigned &gt; Table** list, locate the user you want to remove
2. Select **Remove from role** for that user
3. Accept the confirmation prompt to **Remove**

<Info>The assigned users list updates immediately and the removed user will no longer appear.</Info>

---

### View the Competence Matrix for a Job Role

1. Navigate to **Competency &gt; Job Roles**
2. Select a Job Role that has at least one user in the Assigned list
3. Select the **Matrix** view

<Info>The Matrix view displays a grid of all assigned users alongside the competence requirements for that Job Role. Use this view as your starting point for recording compliance details.</Info>

---

### Set Compliance Details – Registrations

1. From the Matrix view, find a **Registration** requirement (identified by the small grey label beneath the requirement name)
2. Click the grid cell for that requirement for the relevant user
3. In the details panel, enter a **Registration ID** and set the **Expiry Date**
4. Select **Save**

<Info>After saving, the Matrix updates to display the expiry date, with an icon indicating whether the Registration type has an expiry.</Info>

---

### Set Compliance Details – Schemes

1. From the Matrix view, find a **Scheme** requirement (identified by the small grey label beneath the requirement name)
2. Click the grid cell for that requirement for the relevant user
3. In the details panel, set the **Expiry Date**
4. Select **Create**

<Info>After saving, the Matrix updates to display the expiry date, with an icon indicating whether the Scheme type has an expiry.</Info>

---

### Set Compliance Details – Categories

1. From the Matrix view, find a **Category** requirement (identified by the small grey label beneath the requirement name)
2. Click the grid cell for that requirement for the relevant user
3. In the details panel, set the **Expiry Date**
4. Select **Create**

<Info>After saving, the Matrix updates to display the expiry date, with an icon indicating whether the Category type has an expiry.</Info>

---

### Set Compliance Details – Training

1. From the Matrix view, find a **Training** requirement (identified by the small grey label beneath the requirement name)
2. Click the grid cell for that requirement for the relevant user
3. In the details panel, set the **Completed Date**
4. In the **Evidence** panel, select **Choose Files** and locate the certificate file issued to the user on completion of their training course
5. Select **Create**

<Info>After saving, the evidence file is uploaded and the Matrix updates to display the Completed date.</Info>

---

### Set Compliance Details – Inductions

1. From the Matrix view, find an **Induction** requirement (identified by the small grey label beneath the requirement name)
2. Click the grid cell for that requirement for the relevant user
3. In the details panel, set the **Completed Date**
4. In the **Evidence** panel, select **Choose Files** and locate the relevant induction document for that user
5. Select **Create**

<Info>After saving, the evidence file is uploaded and the Matrix updates to display the Completed date.</Info>

---

### Set Compliance Details – Assessments

1. From the Matrix view, find an **Assessment** requirement (identified by the small grey label beneath the requirement name)
2. Click the grid cell for that requirement for the relevant user
3. In the details panel:
   - Set the **Outcome** to **Passed**
   - Set the **Assessment Date**
   - Select an **Assessor**
4. In the **Evidence** panel, select **Choose Files** and locate the CAS Assessment Document for this user
5. Select **Create**

<Info>After saving, the evidence file is uploaded and the Matrix updates to display the expiry date, with an icon indicating whether the Assessment type has an expiry.</Info>

---

### Complete All Required Competence Requirements

1. From the Matrix view, continue recording dates and relevant details for each **Required** competence requirement, following the same steps as the individual requirement types above

<Info>
As each requirement is satisfied:
- The cell colour changes from **Red** to **Green**
- As each group of requirements is completed, the group icon changes from a cross to a tick
- When all Required items are satisfied, the user's overall status changes from **Not Compliant** to **Compliant**

Items in the **Optional** section of the matrix do not affect a user's compliance status.
</Info>

---

### Record an Exemption – Does Not Undertake Activity

1. From the Matrix view, click the coloured cell for a **Training**, **Induction**, or **Assessment** requirement to open the details panel
2. Select **Mark as exemption**
3. For **Exemption Type**, select **Does not undertake activity**
4. Select **Create**

<Info>The requirement is saved and the grid cell displays **DNUA** with a hatched background to indicate the exemption.</Info>

---

### Record an Exemption – Expected First Assessment

1. From the Matrix view, click the coloured cell for a **Training**, **Induction**, or **Assessment** requirement to open the details panel
2. Select **Mark as exemption**
3. For **Exemption Type**, select **Expected first assessment**
4. Set a future date for the **Expected first assessment**
5. Select **Create**

<Info>The requirement is saved and the grid cell displays the Expected first assessment date with a hatched background.</Info>

---

## What To Expect

After following this guide, you should be able to:

- Assign and remove workforce members from Job Roles
- Navigate and use the Competence Matrix view
- Record compliance details for all requirement types: Registrations, Schemes, Categories, Training, Inductions, and Assessments
- Upload evidence certificates against competence requirements
- Track user progress from Not Compliant to Compliant as requirements are satisfied
- Record exemptions where a user does not undertake an activity or is awaiting their first assessment
