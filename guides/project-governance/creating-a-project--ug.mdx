## Overview

This guide walks you through the process of creating a new scheme (project) in Ctrl Hub and setting it up for operations and contractor access. You'll learn how to create a scheme, add work orders, populate them with operations, and share the project with contractor organisations.

---

## What You'll Accomplish

Following this guide, you'll be able to:

- Create a new scheme (project) to organise your work orders
- Create and configure work orders within your scheme
- Populate work orders with operations from your available properties
- Share your scheme with contractor organisations to enable collaboration

---

## Prerequisites

- You have access to Ctrl Hub with appropriate permissions to create schemes
- You understand the naming convention for your organisation
- You understand the work order coding format and requirements (e.g., "WO-2026-001")
- Properties exist in your system for the postcodes where you want to create operations
- You have access to accurate, full postcodes (typically 6-7 characters including the space: e.g., "RH6 9AY")
- You understand the hierarchy: Schemes contain Work Orders, which contain Operations
- You understand how operation templates work and have access to required templates (e.g., "SGN Service Card")
- Contractor organisations are set up and active in your Ctrl Hub instance
- Operatives are enrolled in your system if you plan to assign them to operations
- You have appropriate permissions to share schemes with contractor organisations
- You are familiar with the contractor organisations you will be sharing schemes with

---

## Step-by-Step Guide

### Create a New Scheme

1. Navigate to **Governance > Schemes**
2. Select **Create a new Scheme**
3. In the Create Scheme window, enter a descriptive name for your scheme, for example: "SGN GD3 - Main Route" or "Network Upgrade Phase 2"
4. Select **Create** to proceed

### Create a Work Order

1. Once the Scheme page opens, select **Create a new Work Order**
2. Enter a descriptive name that includes the location, for example: "Balcolme Road, Horley, RH6 9AY" or "Oxford Street Segment A"
3. Enter a unique code for the work order, for example: "WO-2026-001" or "SEO15242-A"
4. Select **Create** to finish

### Add Operations to Your Work Order

1. On the Work Order page, select **create multiple operations** (the link below the map)
2. When the Property search page opens, enter a full postcode and select **Search** (wait a moment for results to load)
3. From the available properties, select one or more properties and select **Next**
4. When prompted, in the Template field, select **SGN Service Card**
5. Select **Finish** to create your operations

### Share the Scheme with a Contractor Organisation

1. Go to **Governance > Schemes** and select the scheme you created
2. Select the **Share** button in the top right of the page
3. If the scheme is available to share, select **Share** next to the contractor organisation name (e.g., "WCB Utilities Sandbox")
4. Select **Create Share** to confirm the sharing

### Assign Operatives to Operations

1. Go to **Governance > Schemes** and select a scheme (such as "SGN Sandbox" if it has been shared with your organisation)
2. Select a Work Order from the scheme
3. When the Work Order record opens, select an Operation from the list
4. When the Operation record opens, tick the checkbox next to the operation you want to assign
5. In the **Actions** menu (at the top of the Operations checkbox column), select **Add**
6. In the **Add Assignees** window, enter an operative's name in the Search field and select **Search**
7. From the filtered list of names, select the operative you want to assign
8. Select **Add Assignees** to confirm the assignment

### Update the Progress Status of an Operation

1. Go to **Governance > Schemes** and select a scheme (such as "SGN Sandbox")
2. Select a Work Order from the scheme
3. From the Work Order page, select an Operation from the list below the map
4. In the left-hand panel, in the **Status** section at the top, select the cog symbol
5. In the **Manage Status** window, select **In Progress**
6. Select **Update Status** to save the change

---

## Expected Results

After completing these steps, you should have:

- A new scheme visible in your Governance > Schemes list
- At least one work order within the scheme with a valid name and code
- Operations created from available properties in the work order
- The scheme shared with the selected contractor organisation, giving them access to view and work on the operations
- Operatives successfully assigned to operations, visible in the Assigned columns and the Operatives section on the Operation page
- Operation status updated to "In Progress", with the operation now appearing in the In Progress column on the Work Order page

## Tips & Best Practices

<Tip>
  Use consistent, descriptive naming conventions for your schemes and work orders. Include location references and sequential numbering (like "WO-2026-001") to make items easy to find and track across
  your organisation.
</Tip>

<Tip>Document your naming scheme in your team's procedures to ensure consistency across all users creating work orders and schemes. This helps maintain a searchable, organised system.</Tip>

<Tip>Build your scheme structure in order: Create the scheme first, then work orders, then add operations. This ensures the proper hierarchy is in place before sharing with contractors.</Tip>

<Tip>Use full, accurate postcodes when searching for properties (typically 6-7 characters including the space, e.g., "RH6 9AY"). Incomplete postcodes will return no results.</Tip>

<Tip>
  Share your scheme with contractor organisations only after creating and configuring the operations you want them to work on. Operatives from the contractor organisation will only see operations in
  shared schemes.
</Tip>

<Tip>Plan your scheme structure before creating work orders and operations. Consider how many work orders you need and which properties belong in each one, rather than creating as you go.</Tip>

<Tip>Verify scheme creation was successful by navigating to Governance > Schemes and confirming the scheme appears in the list before adding work orders.</Tip>

<Tip>When searching for properties, always double-check the postcode is complete and correctly formatted. Incomplete or incorrect postcodes are the most common reason property searches fail.</Tip>

<Tip>
  If you need to add many properties from the same postcode, use the "create multiple operations" batch feature rather than creating each operation individually. This is much faster and more
  efficient.
</Tip>

<Tip>Document your scheme, work order, and operation structure in your project plan. This helps team members find specific work orders and operatives understand project organisation.</Tip>

<Warning>
  When adding operations, the available properties are filtered based on your system access. If you don't see expected properties in the search results, verify that you have the correct postcode and
  appropriate permissions to access those properties.
</Warning>

<Warning>
  Before sharing a scheme with a contractor organisation, ensure that the organisation is active in your Ctrl Hub instance and that you have the appropriate permissions to manage shares. If a scheme
  has already been shared with all available organisations, you'll need to create a new scheme.
</Warning>

## Troubleshooting

| Issue                                                     | Solution                                                                                                                                                                                                     |
| --------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Property search returns no results                        | Ensure you've entered a valid full postcode (typically 6-7 characters like "RH6 9AY"). Wait a moment for the search to complete. Check that you have permissions to access properties in that postcode area. |
| Cannot see Share button or option                         | Verify that you have the appropriate permissions to manage scheme shares. Contact your system administrator if necessary.                                                                                    |
| Work Order creation fails                                 | Ensure the scheme has been fully created and the Scheme page has loaded. Check that you have valid permissions to create work orders within this scheme.                                                     |
| Template field is empty or missing options                | The available templates depend on your system configuration. Ensure the "SGN Service Card" template is available in your instance, or contact your administrator.                                            |
| No contractor organisations appear in Share list          | Ensure contractor organisations are active in your Ctrl Hub instance. Verify that you have permissions to share schemes. Contact your administrator if organisations are missing.                            |
| Operations don't appear in Work Order after creation      | Refresh the Work Order page. The operations may take a moment to appear. Verify the operation creation completed successfully in the background.                                                             |
| Cannot find operative in Add Assignees search             | Ensure the operative is enrolled in your system. Check the spelling of their name. Verify that the operative belongs to the contractor organisation that has access to the scheme.                           |
| Operative assignment fails or doesn't save                | Ensure you followed all steps: tick the operation checkbox, select Add from Actions menu, search for operative, then select Add Assignees. Refresh the page if assignment appears to fail.                   |
| Status cog symbol is not visible                          | The Status section and cog symbol may not be available depending on your user role or the operation state. Verify you have appropriate permissions to update operation status.                               |
| Contractor organisation doesn't have access after sharing | Refresh the contractor's Ctrl Hub instance. The shared scheme may take a moment to appear. Verify the share was successfully created by checking your Governance > Schemes page.                             |
| Postcode search is slow or times out                      | Large postcode areas may take longer to search. Try narrowing your search or breaking the area into smaller postcodes. Check your internet connection.                                                       |
| Cannot assign multiple operatives to one operation        | The Add Assignees function may only allow one operative at a time. Repeat the assignment process for each operative you want to assign.                                                                      |
| Scheme doesn't appear in Governance > Schemes list        | Refresh the Governance page. The new scheme should appear within seconds. Verify you have permissions to view schemes.                                                                                       |
| Work Order appears but operations don't load              | Ensure the Work Order page has fully loaded. Refresh the page or wait a moment for operations to populate. Check that operations were successfully created.                                                  |
| Cannot create Work Order after scheme creation            | Wait a moment for the scheme to fully initialise. If the Scheme page hasn't fully loaded, you won't see the "Create a new Work Order" button. Refresh the page.                                              |
| Work Order code format is rejected or shows error         | Ensure your code format matches your organisation's standards (typically alphanumeric with hyphens). Avoid special characters except hyphens and underscores.                                                |
| Create Scheme button is disabled or not working           | Verify you've entered a scheme name. Scheme names are required. Check that you have permissions to create schemes.                                                                                           |
| Property search hangs or gets stuck                       | If the search takes more than 10 seconds, stop and try again with a smaller area (different postcode). Large postcode areas with many properties can cause delays.                                           |
| Selected properties don't show in the Next step           | Refresh the Property search page. Multiple property selection may have a lag. Verify properties are highlighted/selected before clicking Next.                                                               |
| Operation status cog symbol appears but change won't save | Ensure you selected a status from the **Manage Status** dropdown. Click **Update Status** to confirm. If it still fails, refresh the Operation page and try again.                                           |
| Operative assignment checkbox not appearing               | The checkbox should appear when you're on the Operation record page. Ensure you've navigated to the correct operation within a work order.                                                                   |
| Add Assignees button is greyed out                        | Ensure you've ticked at least one operation checkbox before accessing the Actions menu. The Add button only appears when operations are selected.                                                            |
| Scheme shared but contractor can't see it                 | Ask the contractor to refresh their Ctrl Hub instance. Shared schemes may take a moment to appear. Verify the share was created successfully from your Governance > Schemes page.                            |
| Work order doesn't appear on scheme after creation        | Refresh the Scheme page. The new work order should appear within seconds. If missing, verify the work order was successfully created.                                                                        |
| Operation status shows as blank or undefined              | Manually update the status by selecting the cog symbol in the Status section. Ensure you follow all steps: select status, then click Update Status.                                                          |
