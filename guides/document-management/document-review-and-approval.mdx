## Overview

This guide explains how to organise your documents within a scheme by creating and managing folders. You'll learn how to create new folders, rename them, upload documents, ensure file name uniqueness, and delete folders when no longer needed.

---

## What You'll Accomplish

Following this guide, you'll be able to:

- Create new folders to organise documents within a scheme
- Rename folders to better reflect their contents
- Upload and manage documents within folders
- Understand how Ctrl Hub handles duplicate file names
- Delete empty folders when they're no longer needed
- Set expiry dates for files to manage document lifecycle
- Move files between folders
- Upload new versions of existing files
- Access and view different versions of a file
- Request reviews from approvers with approval deadlines
- View document review and approval status
- Access document review history and approval logs
- Download files to your local device
- Access shared schemes and documents as a contractor
- View the download history and audit log for files

---

## Prerequisites

- You have access to Ctrl Hub with appropriate permissions to manage documents
- You have at least one scheme created in your Governance area
- You understand the folder structure needed for your documents
- You understand which users have permission to create, modify, and delete documents in your schemes
- You are familiar with file formats you plan to upload (PDF, Word, Excel, images, etc.)
- You understand your organisation's file naming conventions and standards
- You have identified who will be approvers in document review workflows before requesting reviews
- Approvers and reviewers you plan to assign have valid email addresses in the system
- You understand document expiry and how it affects access to expired documents
- You understand the document versioning system and how multiple versions are stored
- You are familiar with your organisation's document retention and lifecycle policies
- For contractor access: You understand which schemes will be shared with you and what documents you can access

## Step-by-Step Guide

### Request Document Review and Set Approval Deadlines

1. In your scheme's **Documents** tab, right-click any document and select **Request review**
2. Leave the Reviewers section empty and in the **Approvers** section, select **Add** to add one or more approvers from the available names (your name is already included as an approver by default)
3. Select **Next**, skip the "deadline for comments" option, and select **Yes** for the "deadline for approval"
4. Set a due date and time for when approval is needed for this document
5. Select **Next** and optionally add a message with context or instructions for the approvers
6. Select **Request Review** to finish

### View Document Review and Approval Status

1. In your scheme's **Documents** tab, locate any document with a Review Status indicator
2. Click on the Review Status text (e.g., "Pending") next to the document to open the Document Review Requests panel
3. Alternatively, right-click the document and select **View review requests** to access the same panel
4. The panel displays the current approval status and which approvers have approved, pending, or rejected the document

### Access Document Review History and Approval Log

1. In your scheme's **Documents** tab, click on the Review Status for a document to open the Document Review Requests panel
2. Or right-click the document and select **View review requests** to open the panel
3. In the Document Review Requests panel, view the complete review history showing all approver decisions, comments, and timestamps
4. This log provides a complete audit trail of the approval process for the document

### Download a File

1. In your scheme's **Documents** tab, right-click any file you want to download
2. Select **Download** from the context menu
3. Your browser will offer you the option to Open or Save the file to your local device

### View Shared Schemes and Documents as a Contractor

1. Go to **Governance > Schemes**
2. Select the tab corresponding to the organisation that has shared their schemes with you (for example, "SGN Sandbox")
3. Browse the shared schemes listed on this page
4. Select any shared scheme to access its documents and related information
5. You can view documents, check approval statuses, and download files as permitted

After following these steps, you should have:

## Tips & Best Practices

<Tip>Create a consistent folder structure that reflects your document workflow. For example, organise by project phase (Design, Approval, Execution) or document type (Contracts, Reports, Forms).</Tip>

<Tip>Use descriptive folder names that are easily searchable. Avoid generic names like "Documents" or "Files" to make navigation clearer for all users.</Tip>

<Tip>
  When uploading multiple versions of a file, establish a consistent naming convention (such as including dates or version numbers in the file name) to make it easy to identify which version is which.
</Tip>

<Tip>Use the Move feature to reorganise your documents across folders rather than creating duplicates. This keeps your document library clean and avoids confusion with multiple copies.</Tip>

<Tip>
  When requesting document reviews, set realistic approval deadlines to give approvers sufficient time to review and comment. Include a message with context or specific instructions to help reviewers
  understand what they're approving.
</Tip>

<Tip>Regularly check the Document Review Requests panel to track approval progress. This helps you follow up with approvers who haven't yet provided their decision.</Tip>

<Tip>If you're a contractor, check the available organisation tabs in Governance > Schemes to see which networks have shared their projects with you. Each tab represents a different organisation's shared schemes.</Tip>

<Tip>Establish clear folder structure standards and document them for your team. Share this structure with all users so everyone organises documents consistently.</Tip>

<Tip>Before requesting document reviews, verify that all approvers have valid email addresses in the system. Approvers won't receive notification emails if their email is missing or incorrect.</Tip>

<Tip>When setting expiry dates, consider your document retention policies and compliance requirements. Set dates well in advance so users have time to archive or replace expiring documents.</Tip>

<Tip>Use file naming conventions that include dates or version numbers (e.g., "ProjectPlan_v2_2026-02-14.pdf"). This makes it easy to identify and manage multiple file versions.</Tip>

<Tip>Before uploading large files, check your internet connection stability. Large file uploads may take several minutes. Don't close your browser until the upload completes.</Tip>

<Tip>Include clear context and instructions when requesting document reviews. Use the message field to explain what approvers should focus on and the deadline importance.</Tip>

<Tip>Regularly archive or clean up old document versions to keep your folder structure manageable. Use the Download History to identify which versions are actively used.</Tip>

<Tip>When multiple users need to edit a document, coordinate who will upload the next version to avoid conflicting changes. Use version control discipline with meaningful file names.</Tip>

<Warning>You can only delete empty folders. If a folder contains documents, you must move or delete the documents first before you can remove the folder.</Warning>

<Warning>When uploading multiple versions of the same file, pay attention to the automatic numbering (1), (2), etc. to ensure you're working with the correct version.</Warning>
