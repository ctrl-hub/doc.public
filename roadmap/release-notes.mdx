## Release Schedule

<Card>
  New releases are typically deployed on the following schedule:

- **Major versions** (e.g., 3.0.0) — Every 6 months
- **Minor versions** (e.g., 3.2.0) — Every 4-6 weeks
- **Patch releases** (e.g., 3.2.1) — As needed for critical fixes

We announce releases via email and in-app notifications.

</Card>

## Upgrade Paths

### Upgrading from Version 2.x to Version 3.x

<Card highlight="info">Upgrading to version 3 introduces several breaking changes. Review the migration guide before upgrading.</Card>

| Area               | Change                         | Action Required                       |
| ------------------ | ------------------------------ | ------------------------------------- |
| **API Endpoints**  | Some endpoints deprecated      | Update integrations to new endpoints  |
| **Authentication** | Token format changed           | Update stored credentials             |
| **Data Format**    | Some field structures modified | Validate existing data in new version |

[Full Migration Guide →](/guides/)

### Upgrading within Version 3.x

Minor and patch updates within version 3 are backwards compatible and can be applied without code changes.
